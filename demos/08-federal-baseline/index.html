08 基线 — 联邦宪法：不可变边界变更流

来源：intent-schema-compiler 中的 immutable_boundaries。演示 Breaking Change 的委员会审批流程。

📜 当前联邦宪法（v1.0.0）

语义层
status.critical
系统级故障，需立即响应。映射红色脉冲 + 高优提示音。
⚠️ immutable: true — 变更必须发新版本
治理层
destructive-action
高危操作意图。必须二次确认，AI 禁止直接执行。
⚠️ immutable: true — 变更必须发新版本
治理层
human-ai-boundary
人机边界划分。human_mandatory + ai_prohibited 列表。
⚠️ immutable: true — 变更必须发新版本
语义层
status.warning
潜在风险，需关注。映射黄色静态令牌。
✓ immutable: false — 兼容变更可原地更新

兼容变更 vs Breaking Change 对比

✅ 兼容变更（Normal）
  • 新增语义令牌（不修改现有）
  • 扩展同义词映射白名单
  • 增加场景测试用例
  • Steward 审批即可，无需委员会
  • 原地更新，不发新版本
❌ Breaking Change（不可变边界变更）
  • 修改 immutable: true 的令牌定义
  • 删除或重命名语义令牌
  • 收紧 ai_prohibited 列表
  • 必须联邦委员会 3/5 投票通过
  • 旧版本冻结，强制发新版本
