🔷 控制平面

intent-schema-compiler
全局唯一语义事实源，设计意图的"联邦宪法"。<

三层结构：
• 语义层：语义令牌 + 同义词防火墙
• 治理层：不可变边界 + 人机边界矩阵
• 执行层：场景测试 + 编译思维链

验证原则：声明式、版本锁定

📋 Registry

语义注册表
管理语义 ID 全局唯一注册、版本化与兼容性分析。

核心能力：
• SemVer 版本管理
• 影响面分析（Git Diff → 下游定位）<
• 产品绑定配置（bindings/*.yaml）

⚙️ Compiler

契约编译器
将 YAML 意图协议翻译为各层可执行产物。

三阶段编译：
• 意图加载 + 语义解析
• 约束推导 + 产物生成
• 场景验证 + 影响分析

验证原则：声明式

🛡️ Validator

四层推演引擎
内容进入生产链路前的分层安检。

四层推演：
• T1 语法：结构完整性 → BLOCK
• T2 语义：令牌匹配 → BLOCK
• T3 安全：禁止模式 → BLOCK
• T4 美感：可读性 → WARN

验证原则：阻断优于修正

🚀 Runtime

运行时拦截
将编译产物部署到生产环境，现场执行拦截。

三种守卫：
• React HOC：withIntentGuard
• API 中间件：apiGuardMiddleware
• LLM Tool Guard：toolGuard

验证原则：正交穿透

👁️ Bridge

观测桥接器
采集运行时漂移，反向修正控制平面。

闭环流程：
• 采集：LoongSuite / OTel Trace
• 归因：根因定位到契约规则
• 反哺：自动创建 Registry PR

验证原则：版本锁定

🟣 基础设施层

纵向生产链路
现有业务基础设施，承载用户请求与业务逻辑。

五层穿透：
• L1 Token：Ant Design / Carbon
• L2 组件：React / Vue / Angular
• L3 API：OpenAPI / GraphQL
• L4 容器：K8s / Envoy / Docker
• L5 DB / LLM：MySQL / PostgreSQL / GPT

治理方式：网格化覆盖，不替代只注入
