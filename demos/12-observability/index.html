12 与可观测平台的关系

LoongSuite GenAI SemConv × Schema-As-Code：运行时观测与设计时约束的阶段互补与闭环咬合

🔵 运行时观测（LoongSuite）

  • 时机：漂移发生后（事后）
  • 问题："漂移发生了吗？"
  • 手段：采集、追踪、分析
  • 产出：Trace / Metric / Log
  • 消费方：SRE、运维、算法工程师

🔶 设计时约束（Schema-As-Code）

  • 时机：漂移发生前（事前）
  • 问题："漂移应该被允许吗？"
  • 手段：约束、校验、拦截
  • 产出：Intent Protocol / Constraint Artifact
  • 消费方：设计师、前端、AI 工程师

♻️ 闭环咬合演示：从观测到约束

1
运行时观测：LoongSuite 采集异常 Trace

当 LLM temperature > 0.8 时，同义词替代概率显著上升。LoongSuite SemConv 的 Token 级观测捕获到异常。

[Entry Span] invoke_skill: alert_card_generation
[Step Span] ReAct iteration #3
[Token] per_token_time: 45ms (baseline 20ms)
[Drift] alert_level: "严重" → synonym substitution detected
[Drift] confidence: 0.95 → temperature correlation: 0.87
2
归因引擎：Bridge 定位根因

Bridge 消费 LoongSuite 的 Trace 数据，语义归一化后定位到控制平面的具体规则。

[Normalizer] mapping: "严重" → status.critical (failed)
[RootCause] synonym-mapping.yaml: "严重" confidence_threshold: 0.95
[RootCause] LLM config: temperature=0.9, top_p=0.95
[Impact] 3 products affected, 12 components impacted
3
自动反哺：Bridge 创建 Registry PR

归因结果自动转化为控制平面的规则收紧补丁，提交 PR 等待联邦委员会审批。

🤖 [Auto] 收紧"严重"同义词规则 + 降低 LLM temperature 阈值
文件: schema/v1.1.0/semantic/synonym-mapping.yaml
变更: confidence_threshold 0.95 → 0.85
新增: llm_config.max_temperature: 0.8
4
设计时更新：Compiler 重新编译

PR Merge 后，Compiler 自动重新编译约束产物，分发到五层基础设施。

编译前 (v1.0.0)
synonym: "严重"
threshold: 0.95
action: warn
编译后 (v1.1.0)
synonym: "严重"
threshold: 0.85
action: block
llm_max_temp: 0.8
5
下一周期验证：观测闭环效果

新规则生效后，下一运行周期中 Validator 的拦截率提升，Bridge 的漂移告警下降。

[Metric] semantic_consistency_score: 78% → 94%
[Metric] rule_interception_rate: 62% → 97%
[Metric] human_escalation_ratio: 35% → 8%
[Alert] synonym_drift: RESOLVED
价值公式：运行时观测成本 + 设计时约束成本 < 漂移修复成本 + 舆情处理成本
LoongSuite 回答"漂移发生了吗"，Schema-As-Code 回答"漂移应该被允许吗"。两者咬合，才是完整的语义治理。
